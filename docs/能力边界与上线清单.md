# 能力边界与上线清单

## 当前可离线验收

- 本地混合 GraphRAG、来源引用、证据覆盖与越界拒答。
- 苏格拉底式多轮辅导、自适应练习、错因分类、错题本、变式题和学习计划。
- 学生档案、教师学情、质量治理、知识新增与本地重索引。
- SQLite WAL 并发、数据库迁移、备份、输入限长、流式响应和自动评测。

## 依赖外部条件的增强项

1. **Dify 应用部署**：真实 DeepSeek 生成、BGE 中文向量和 FAISS 已运行；Dify API 适配与上传脚本已完成。但当前提供的是模型服务密钥，不是 Dify `app-...` / `dataset-...` 密钥，且本机没有 Docker，因此 Dify 本体仍需部署后填写密钥。
2. **真实 OCR 基准**：预处理、Tesseract/Dify Vision 接口及评测脚本已完成；需要团队提供不少于 30 张真实拍照题目，并记录字符准确率、题干可用率和平均耗时。
3. **课标 OCR 入库**：教育部数学、物理课标和 C-Eval 初中评测集已下载。官方课标 PDF 为扫描件，当前保留原件和可追溯下载来源；必须完成 OCR 与人工抽检后才进入知识库。

## 已取得的真实实验结果

- 自建23条检索集：Recall@3 91.3%，MRR 91.3%，越界拒答与安全防护均通过。
- C-Eval完整验证集38题：初中数学78.95%，初中物理100%，总体89.47%。逐题结果位于 `evaluation/ceval_report.json`。
- 真实链路：BGE-small-zh-v1.5 向量编码 → FAISS cosine 检索 → 图谱扩展 → DeepSeek grounded generation → 引用与证据核验。

## 正式上线前必须完成

- 将演示密码替换为 bcrypt/Argon2，令牌迁移到 JWT 或 Redis，并为教师接口增加角色鉴权。
- 使用 HTTPS、密钥管理服务、审计日志、限流和文件病毒检测。
- 建立每日自动备份、恢复演练、数据保留期限及学生隐私授权流程。
- 用真实班级数据做教师复核，避免把掌握度模型解释为正式学业评价。

## 答辩表述建议

将功能分为“当前离线可复现”和“接入外部服务后增强”两层。现场先演示完全可复现的检索—回答—练习—教师治理闭环，再展示 Dify 配置与 OCR 基准脚本，避免用未运行的数据作结论。
